Итоговый проект «Система аналитики личных трат»
Цель проекта — разработать микросервисное приложение для учёта и анализа личных финансов. Приложение позволит фиксировать личные траты и бюджеты, получать базовые отчёты и использовать Google Таблицы как UI, взаимодействуя с системой через API.

Функциональные требования

Учёт транзакций.

Система должна позволять добавлять финансовые транзакции (сумма, категория, описание, дата) и хранить их в базе данных.

Управление бюджетами.

Упользователя должна быть возможность устанавливать бюджеты по категориям расходов. При добавлении транзакции система должна проверить, не превышен ли лимит бюджета.

Импорт и экспорт данных.

Система должна уметь загружать транзакции из CSV-файлов (например, выгрузок из банковских приложений) и выгружать данные в CSV для анализа.

Генерация отчётов.

Система должна формировать сводные отчёты по расходам за определённый период: траты по категориям, процент выполнения бюджета и общий итог расходов.

API-доступ из Google Таблиц.

Google Таблицы необходимо использовать как основной интерфейс: пользователь добавляет транзакции или бюджеты через таблицу, а скрипты Google Apps Script отправляют запросы в API проекта. В обратную сторону система может заполнять таблицу данными отчётов.

Безопасность (JWT).

Доступ к API должен быть защищён с помощью JWT-токенов. Все запросы к системе должны требовать авторизации.

Технические требования

Язык разработки — Go.
Формат взаимодействия микросервисов — gRPC.
HTTP API для Google Таблиц и внешних клиентов — Gateway.
Хранение данных — PostgreSQL, Redis.
UI — Google Таблицы (Google Apps Script для интеграции).
Проектдолжен быть разработан по принципам чистой архитектуры (clean architecture) — чёткое разделение слоёв.
Развёртывание локально и предоставление API по публичному адресу — Docker Compose.
Предполагаемый результат

Готовый проект должен представлять собой микросервисное приложение, доступное через HTTP API.

Структура проекта

Проект должен быть структурирован как микросервисное приложение и состоять из компонентов:

Gateway:
принимает HTTP-запросы от Google Таблиц;
валидирует данные, проверяет JWT, преобразует запросы в gRPC-вызовы;
отвечает клиенту результатом работы Ledger.
Ledger — основной бизнес-сервис:
хранит транзакции и бюджеты в PostgreSQL;
реализует бизнес-логику: добавление транзакций, проверка бюджета, генерация отчётов, импорт и экспорт CSV.
Auth — регистрация и логин пользователей:
генерирует JWT-токены для доступа к API.
Google Таблицы (UI):
интерфейс для пользователей;
пользователь вносит данные (транзакции, бюджеты) напрямую в таблицу;
Google Apps Script автоматически вызывает публичное API Gateway для записи данных;
отчёты от Ledger могут выгружаться обратно в таблицу.
